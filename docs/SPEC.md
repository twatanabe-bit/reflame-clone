## 1. エグゼクティブサマリー

**ShukatsuOS**は、日本の就活市場における情報の断片化・手入力の煩雑さ・精神的不安を
テクノロジーで根本から解決する、次世代の就活統合管理プラットフォームである。

### バリュープロポジション
> 「メール・LINE・各社マイページ・ES・面接対策・OB訪問 — すべてが一つになる。手入力ゼロの就活OS。」

## 2. 競合比較と差別化戦略

### 2.1 国内主要サービスの課題分析

| サービス | 主な弱点 |
|---|---|
| **リクナビ** | UIが旧世代、通知過多、情報が企業寄り、手入力が膨大 |
| **マイナビ** | 機能肥大化によるUX劣化、パーソナライズ不足 |
| **ONE CAREER** | ES・体験談は強いが管理機能が弱い、モバイル体験が中途半端 |
| **外資就活** | ターゲットが狭い（上位校限定感）、コミュニティ頼み |
| **OfferBox** | 逆求人特化で総合管理不可、ES管理なし |
| **Lognavi** | 動画就活はニッチ、企業数限定的 |
| **Notion（自作管理）** | 設定が面倒、就活特化の自動化なし、テンプレ依存 |

### 2.2 ShukatsuOSの圧倒的差別化

| 差別化軸 | 実現方法 |
|---|---|
| **手入力ゼロ** | メール自動解析AI + 各社マイページ連携による自動データ取得 |
| **情報一元化** | メール / LINE / カレンダー / ES / 面接メモを単一ダッシュボードに統合 |
| **不安解消** | RPG風ステータス画面 + AIメンター + 進捗の完全可視化 |
| **Z世代UX** | Liquid Glass デザイン + 60fps操作 + ゲーミフィケーション |
| **AI-Native** | ES添削 / 面接練習 / 企業マッチング / 締切自動管理をAIが実行 |

## 3. コア機能（Feature Set）

### 3.1 AI自動同期エンジン（SyncEngine）

#### メール自動解析
- Gmail / Outlook連携でメールを自動スキャン
- AI（Claude API）が就活関連メールを自動分類
  - 選考案内 → 選考ステータス更新
  - 説明会案内 → カレンダー自動登録
  - 合否通知 → ステータス即時反映 + 通知
  - ES締切 → デッドライン自動設定

#### マイページ連携
- 主要企業のマイページをブラウザ拡張で自動ログイン・情報取得
- セキュアなCredential Vault（E2E暗号化）でID/PASS管理
- 連携企業: リクナビ / マイナビ / ONE CAREER / 各企業独自マイページ

#### カレンダー統合
- Google Calendar / Apple Calendar 双方向同期
- 面接・ES締切・説明会を自動カラーコード分類
- 移動時間の自動算出（Google Maps API連携）

### 3.2 ES 2.0（Entry Sheet Management）

#### AI添削エンジン
- 過去の合格ES 50万件のデータベースを学習したAIモデル
- 企業×設問パターン別の最適な構成を提案
- 「ガクチカ」「志望動機」「自己PR」の3大テンプレートを自動生成
- フィードバックは5段階: 構成 / 具体性 / 論理性 / 独自性 / 文字数最適化

#### 一括管理 & 使い回し最適化
- 全企業のESを一覧管理（締切順 / 企業順 / ステータス順）
- 類似設問の自動検出 → 過去回答の転用提案
- 企業ごとの「刺さるキーワード」をAIが自動抽出
- バージョン管理（Git的な差分表示）

#### 提出トラッキング
- 提出済み / 下書き / 未着手のステータス管理
- 締切までの残日数アラート（3日前 / 1日前 / 3時間前）
- 提出後の選考進捗と連動

### 3.3 リアルタイム・ダッシュボード

#### 選考パイプライン
- カンバンボード形式: ES提出 → 書類通過 → 1次 → 2次 → 最終 → 内定
- ドラッグ&ドロップで手動ステータス変更も可能
- 各ステージの通過率をリアルタイム算出
- 同期の平均と比較（匿名統計）

#### 統計ダッシュボード
- エントリー数 / ES提出率 / 書類通過率 / 最終面接率 / 内定率
- 業界別・企業規模別の分析
- 週次/月次のアクティビティヒートマップ

### 3.4 OB/OG・内定者DB連携

#### マッチング機能
- 志望企業 × 大学 × 学部でOB/OGを自動検索
- 内定者のES・面接体験談を閲覧
- チャットベースでの相談（匿名オプション付き）

#### AIアドバイザー（ShukatsuMentor）
- 24時間対応のAIメンター
- 面接質問の回答練習（音声対応）
- メンタルヘルスチェック & 励ましメッセージ
- 企業研究の要約レポート自動生成

### 3.5 オフライン/爆速UI

#### パフォーマンス要件
- First Contentful Paint: < 0.8秒
- Time to Interactive: < 1.5秒
- オフライン対応: Service Worker + IndexedDBでローカルキャッシュ
- Optimistic UI: 操作は即時反映、バックグラウンドで同期

#### インタラクション設計
- 全アニメーション60fps保証（GPU accelerated）
- スワイプジェスチャー対応（モバイル）
- キーボードショートカット完備（デスクトップ）
- ダークモード / ライトモード自動切り替え

## 4. ユーザー体験（UX）設計

### 4.1 黄金の1分間フロー

```
0s    アプリ起動
      ↓
3s    ダッシュボード表示
      「今日やるべきこと」が3つ以内で表示
      ↓
10s   最も緊急なタスクをタップ
      （例: 明日締切のES → ES編集画面へ）
      ↓
30s   AIが下書きを提案済み → 微修正するだけ
      ↓
45s   ES提出完了 → 達成アニメーション
      ステータスが自動更新
      ↓
60s   次のタスクが表示 or 「今日のミッション完了!」
```

### 4.2 ゲーミフィケーション要素

| 要素 | 実装 |
|---|---|
| **レベルシステム** | アクション（ES提出、面接参加等）でEXP獲得 → レベルアップ |
| **デイリーミッション** | 「今日のES 1社」「企業研究 15分」等の小目標 |
| **バッジコレクション** | 「ES10社マスター」「面接王」等の実績バッジ |
| **ストリーク** | 連続ログイン日数 → 特別称号獲得 |
| **ランキング** | 匿名のアクティビティランキング（オプトイン） |
| **進化するアバター** | 就活の進捗に応じてアバターが成長 |

### 4.3 メンタルケア機能

- 週1回の「気持ちチェックイン」（5段階スケール）
- お祈りメール受信時の自動フォロー（AIからの励まし + 次のアクション提案）
- 「休憩しよう」リマインダー（連続作業2時間で発動）
- 同じ経験をした先輩の匿名体験談表示

## 5. 技術スタック & セキュリティ

### 5.1 技術構成

| レイヤー | 技術 | 選定理由 |
|---|---|---|
| **Frontend** | Next.js 16 + React 19 + TypeScript | RSC + Server Actions で爆速、既存基盤活用 |
| **Styling** | Tailwind CSS 4 + Framer Motion | Liquid Glass デザイン実現、GPU accelerated animation |
| **State** | Zustand + TanStack Query | 軽量 + キャッシュ最適化 |
| **Backend** | Next.js API Routes + Hono (Edge) | Edge Runtime で低レイテンシ |
| **Database** | Supabase (PostgreSQL) + Drizzle ORM | リアルタイム同期 + 型安全 |
| **AI** | Claude API (Opus/Sonnet) | ES添削・メール解析・メンター機能 |
| **Auth** | Supabase Auth + Passkey | パスワードレス認証 |
| **Storage** | Supabase Storage + R2 | ES添付ファイル・アバター画像 |
| **Search** | Meilisearch | 爆速全文検索（企業・ES・体験談） |
| **Monitoring** | Sentry + Vercel Analytics | エラー追跡 + パフォーマンス監視 |
| **Deploy** | Vercel (Edge) | グローバルCDN + ISR |

### 5.2 セキュリティ要件

| 要件 | 実装 |
|---|---|
| **データ暗号化** | AES-256-GCM（保存時）+ TLS 1.3（通信時） |
| **認証** | Passkey (WebAuthn) + OAuth 2.0 PKCE |
| **認可** | RBAC + Row Level Security (Supabase RLS) |
| **Credential Vault** | E2E暗号化、サーバー側で復号不可 |
| **個人情報保護** | APPI（個人情報保護法）完全準拠 |
| **監査ログ** | 全操作の不可逆ログ記録 |
| **脆弱性対策** | OWASP Top 10対応、定期ペネトレーションテスト |
| **データ削除権** | ユーザーが全データを即時完全削除可能 |
| **SOC 2 Type II** | Phase 3で取得予定 |

## 6. ロードマップ

### Phase 1: MVP（3ヶ月）— 2026年Q2
**目標**: コアユーザー1,000人獲得

- ダッシュボード（選考パイプライン + カレンダー）
- ES管理（CRUD + AI添削基本版）
- メール自動解析（Gmail連携）
- RPGステータス画面（基本版）
- 認証（Google OAuth + Passkey）

### Phase 2: Growth（3ヶ月）— 2026年Q3
**目標**: ユーザー10,000人、DAU 30%

- LINE連携
- OB/OG・内定者DB
- AIメンター（チャット）
- ゲーミフィケーション全機能
- マイページ連携（主要10社）
- オフライン対応

### Phase 3: Scale（6ヶ月）— 2026年Q4〜2027年Q1
**目標**: ユーザー100,000人、就活アプリシェア1位

- 面接練習AI（音声 + 動画分析）
- 企業マッチングAI
- メンタルケア機能
- ブラウザ拡張（マイページ自動連携）
- SOC 2認証取得
- 大学キャリアセンター向けB2B版

### Phase 4: Domination（継続）— 2027年〜
**目標**: 日本の就活インフラに

- 採用企業向けダッシュボード（B2B収益化）
