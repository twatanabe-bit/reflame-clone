# Part 6｜ブランドガバナンス・運用ルール

ブランドは「決めた瞬間」ではなく「運用された期間」によって価値が定まる。本パートは、REFLAMEブランドを **日々の判断と発信物の品質** に落とし込むための運用設計。

---

## 6.2 ブランドチェックリスト（全発信物の必須項目）【提案】

### 6.2.1 共通チェック項目（全発信物）

- [ ] 発信物のトンマナがREFLAMEの「誠実×知的×伴走的」と一致しているか
- [ ] 「変化」「生存実感」「君の言葉で」「武器」のいずれかブランドワードを含むか
- [ ] 数字・データ・固有名詞のいずれかでファクトを示しているか
- [ ] 不適切表現（誇大表現、保証表現、競合批判、個人攻撃）を含んでいないか
- [ ] 個人情報・本人同意の取り扱いが適切か
- [ ] ブランドガイドラインのVI規定（色・フォント・ロゴ）に準拠しているか
- [ ] CTAが1つに絞られているか（複数CTAは離脱増）

### 6.2.2 媒体別追加チェック

#### YouTube動画

- [ ] 冒頭5秒に「変化を、あなたの武器に。」（中途）等のタグライン露出があるか
- [ ] サムネイルに「顔・数字・刺激的単語」の3要素があるか
- [ ] タイトルに「数字 + 変化動詞 + 対象者属性」のテンプレが反映されているか
- [ ] CTAが統一フォーマットになっているか

#### TikTok動画

- [ ] フックが2秒以内で成立しているか
- [ ] 月次の投稿カレンダー（事業部別）と整合しているか
- [ ] 出演者の本人合意・公開可否確認があるか

#### スカウト文面

- [ ] 件名が35文字以内・ペルソナに対応しているか
- [ ] 冒頭3行に「相手レジュメへの言及」があるか
- [ ] RtB（Salesforce事例化等）が1文含まれているか
- [ ] CTAが「15分の面談」など低ハードルか

#### 法人向け資料

- [ ] 冒頭にAI関連の権威性（Agentforce / SFUG等）が記載されているか
- [ ] 「採用を、科学する」のサブタグライン（建築）が反映されているか
- [ ] 数値・実績・事例が具体的に示されているか

#### MAメール

- [ ] 件名35文字以内、本文1,000文字以内
- [ ] 冒頭3行に状況理解を示しているか
- [ ] CTAが1つに絞られているか

#### LINE配信

- [ ] 1メッセージ300文字以内
- [ ] 絵文字は使うが、過剰でないか
- [ ] 配信頻度が週2回以下に収まっているか

## 6.4 VIガイドライン（ビジュアル）【要確認】

### 6.4.1 ロゴ運用

【要確認】現状のREFLAMEロゴの正式バージョン・カラーコード・最小サイズ・余白規定を別途整理。

### 6.4.2 カラーパレット【要確認】

| 役割 | カラーコード（仮） | 用途 |
|---|---|---|
| プライマリ | （要確認） | ロゴ・主要見出し・CTA |
| セカンダリ | （要確認） | サブ要素・装飾 |
| アクセント | （要確認） | 強調・限定的使用 |
| グレースケール | グレー5階調 | 本文・補助 |

### 6.4.3 タイポグラフィ

- 和文：Noto Sans JP（既存採用、layout.tsxに記載あり）
- 英文：Inter または Roboto
- ウェイト：Regular（400） / Bold（700） / Black（900）
- 見出し最小：18px
- 本文最小：14px

### 6.4.4 写真・イラスト

- 写真：ドキュメンタリーテイスト優先、スタジオ撮影は最小限
- イラスト：データ可視化を優先、抽象表現は限定的に
- アイコン：Lucide Icons（既存採用）

## 6.5 危機管理（ブランドクライシス対応）【提案】

### 6.5.1 想定リスクシナリオ

| シナリオ | 想定インパクト | 即時対応 |
|---|---|---|
| SNS炎上（候補者発信での苦情拡散） | 中〜高 | 6時間以内に事実関係確認・本人謝罪・再発防止公表 |
| 候補者情報漏洩 | 高 | 24時間以内に該当者通知・原因究明・外部報告 |
| 元社員からの内部告発 | 中 | 事実確認・労務責任者対応・必要時の公式コメント |
| AI誤判定によるミスマッチクレーム | 中 | 個別補償＋AI運用ルール見直し公表 |
| 競合からの誹謗中傷 | 低 | 基本無視（反応せず）。証拠保全のみ |
| 経営層の不適切発言 | 高 | 即時謝罪・撤回・経緯公表 |

### 6.5.2 危機対応の意思決定フロー

```
[インシデント発生]
   ↓
[一次対応者（マーケ実行 or 各事業部長）が事実確認]
   ↓
[CMOへエスカレーション（30分以内）]
   ↓
[CMO判断：軽微 / 中程度 / 重大]
   ↓
   ├─ 軽微：CMO判断で対応
   ├─ 中程度：CMO + 代表で対応
   └─ 重大：経営会議＋顧問緊急招集
   ↓
[対応実行＋48時間以内に事後レポート]
```

### 6.5.3 平時の予防運用

- 月次：SNS発信物の事後監査（CMO実施）
- 月次：候補者・クライアントからの苦情ログ集計・原因分析
- 四半期：危機対応シミュレーションの実施

## 6.6 社内浸透（インナーブランディング）【提案】

### 6.6.1 浸透施策のステップ

| Step | 内容 | 担当 |
|---|---|---|
| 1. 認知 | 全社員へのブランドキックオフ（オフサイト or 全社総会） | CMO＋CHRO |
| 2. 理解 | ブランドブック配布 + Q&Aセッション | CHRO |
| 3. 体現 | 各事業部長による日常業務への適用ガイド | 各事業部長 |
| 4. 評価 | 半期評価にブランド体現項目を組み込む | CHRO |
| 5. 強化 | ブランド体現事例の月次表彰 | CMO＋CHRO |

### 6.6.2 ブランドブックの構成（提案）

- 第1章：REFLAMEの存在意義（Philosophy）
- 第2章：どこを目指すか（Vision・Position）
- 第3章：何を大切にするか（Values・5原則）
- 第4章：どう話すか（言葉遣い辞典の社員版）
- 第5章：どう見せるか（VIガイドライン抜粋）
- 第6章：どう判断するか（DO/DON'T事例集）
- 巻末：用語集・FAQ

A5サイズ・40-60ページ程度を想定。電子版（PDF）＋紙版両方。

### 6.6.3 入社時オンボーディングへの組み込み

- 入社1週目：ブランドブック通読・テスト
- 入社2週目：CMOまたは経営層との「REFLAMEを語る」セッション
- 入社1ヶ月目：自分の言葉でREFLAMEを30秒で説明できるか確認

## 6.7 外部委託先（協力会社）との連携【提案】

### 6.7.1 委託先のブランド遵守義務

【既存：業務委託契約書_REFLAME（SNS運用・動画撮影編集）に記載済】

委託先には以下を必須要件として契約に組み込む：
- REFLAMEブランドガイドラインの遵守
- 景品表示法・薬機法・著作権法・ステマ規制等の関連法令遵守
- 公開前のREFLAME側承認フロー遵守

### 6.7.2 委託先のオンボーディング

- 契約時：ブランドブック電子版の提供＋オリエンセッション（1時間）
- 月次：成果物のフィードバック＋ガイドライン違反の即時是正
- 四半期：委託先評価＋契約継続判断

## 6.8 法務・コンプライアンス上の留意事項【提案】

### 6.8.1 個人情報保護

- 候補者情報の取り扱い：APPI（個人情報保護法）完全準拠
- 面談録・SF記録の暗号化保管
- 退職時のアクセス権即時剥奪

### 6.8.2 職業安定法・労働者派遣法

- 人材紹介事業のライセンス遵守
- 求人広告の表記ルール（虚偽記載禁止、明確な労働条件記載）

### 6.8.3 商標・知的財産

- Salesforce / Agentforce / その他協力会社の商標利用は許諾範囲内
- 自社の発信物の著作権を明確化（社員作成物の社内帰属、委託先作成物の取り扱い）

### 6.8.4 ステルスマーケティング規制

- SNS発信での「広告」表記の徹底
- TikTok・Instagramでの委託インフルエンサーとのタイアップ表記
