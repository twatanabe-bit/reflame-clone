# Part 8｜KPI / ダッシュボード設計

ブランド戦略は「測れなければ価値が落ちる」。本パートは、戦略の効果を3層構造で可視化するKPI設計。

---

## 8.1 KPI体系の全体図

```
┌─────────────────────────────────────────────────┐
│  Layer 1: Outcome KPI（事業成果）                  │
│  - 売上 / 計上 / LTV / 営業利益                    │
│  - 候補者離職率 / クライアント継続率                 │
└─────────────────────────────────────────────────┘
                       ▲
                       │
┌─────────────────────────────────────────────────┐
│  Layer 2: Output KPI（ブランド露出・行動）          │
│  - 面談数 / 推薦数 / LINE登録                       │
│  - YouTube/TikTokフォロワー / SEOセッション数        │
│  - 登壇・掲載回数                                  │
└─────────────────────────────────────────────────┘
                       ▲
                       │
┌─────────────────────────────────────────────────┐
│  Layer 3: Brand KPI（認知・想起・好意）             │
│  - 指名検索数 / 候補者想起率                        │
│  - NPS / ブランドリフトスコア                       │
│  - 社員ブランド体現率                              │
└─────────────────────────────────────────────────┘
                       ▲
                       │
┌─────────────────────────────────────────────────┐
│  Layer 4: Activity KPI（日次の活動量）              │
│  - スカウト送付数 / 投稿本数 / メール配信数            │
│  - 顧問定例実施率 / ブランドチェック実施率           │
└─────────────────────────────────────────────────┘
```

各レイヤーは因果関係でつながる：Activity → Output → Brand → Outcome。各レイヤーの遅効性も加味して測る。

## 8.6 ダッシュボード設計【提案】

### 8.6.1 ダッシュボードの3層構造

| ダッシュボード | 対象 | 更新頻度 | 設置場所 |
|---|---|---|---|
| **経営ダッシュボード** | 経営会議・取締役 | 月次 | Notion / Salesforceレポート |
| **マネージャーダッシュボード** | CMO・各事業部長 | 週次 | Notion / Salesforceレポート |
| **オペレーションダッシュボード** | マーケ実行・各CA | 日次 | Salesforce画面 / Slackボット |

### 8.6.2 経営ダッシュボードの構成

```
[1. 売上ハイライト]
   先月実績 / 月次推移 / YoY / 目標達成率

[2. ブランド指標]
   指名検索数 / NPS / メディア掲載数

[3. ファネル全体]
   認知→興味→検討→行動→決定の各段階数

[4. 事業部別サマリ]
   中途・新卒・建築の月次成果

[5. AI活用ブランド資産]
   登壇・掲載・SF連携の進捗

[6. リスク/アラート]
   未達KPI・遅延タスク・危機シグナル
```

### 8.6.3 既存ダッシュボード【既存】

- **3事業部KPI統合DB**（Pipeline B 自動更新、月曜朝8:00）
- **CMO優先課題マップDB**（顧問定例での更新）
- **AOS示唆DB**（青木顧問の示唆ナレッジ）
- **マーケKPIレポート v2**（週次顧問定例用）

これらを **経営ダッシュボードに統合表示** する仕組みを構築（Day 60までに完成）

## 8.7 KPI測定の運用ルール【提案】

### 8.7.1 計測の責任分担

| KPIレイヤー | 計測責任者 | 報告先 | 頻度 |
|---|---|---|---|
| Outcome | CFO / 経理 | 経営会議 | 月次 |
| Output | 各事業部長 | CMO | 週次 |
| Brand | CMO | 経営会議 | 月次 |
| Activity | マーケ実行 | 各事業部長 | 日次 |

### 8.7.2 計測の自動化

- Salesforceダッシュボード経由で自動計測（最優先）
- SNSはAPI連携でAccount Engagement / 自社BIへ
- 手動計測項目は月次の手作業を許容（CMO直轄）

### 8.7.3 計測精度の維持

- 月次のデータクレンジング（既存：RevOps組織設計書に基づく週次クレンジング運用あり）
- 計測定義の文書化（Notionに用語集として保管）
- 定義変更時の社内通知ルール

## 8.8 ブランドリフト調査（年2回）【提案】

### 8.8.1 調査設計

| 項目 | 内容 |
|---|---|
| 調査対象 | 候補者100名（中途・新卒・建築 各30-40名）、採用候補50名、クライアント企業30社 |
| 調査方法 | オンラインアンケート＋一部インタビュー |
| 調査時期 | Day 90、Day 270（半年に1回） |
| 委託先 | 【要確認】調査会社の選定 |

### 8.8.2 主要質問項目

1. REFLAMEの第一想起度（人材紹介でREFLAMEを最初に想起するか）
2. REFLAMEの認知度（知っている／会話で出る／使ったことがある）
3. REFLAMEの好意度（良い印象／普通／悪い印象）
4. REFLAMEの差別化認知（他社と比べて何が違うと思うか）
5. REFLAMEの推奨意向（NPS）
6. REFLAMEの「人とAIの共存」認知（具体的な訴求点が伝わっているか）
7. REFLAMEの「変化を売る」認知（タグラインのメッセージ理解度）

### 8.8.3 出力物

- 経時変化グラフ
- セグメント別ブランド指標
- 競合との認知比較
- 改善優先課題

## 8.9 KPIに連動した報奨制度【提案】

### 8.9.1 個人レベル

- ブランドKPI（NPS, ブランドリフトスコア）達成への貢献を半期評価に反映
- 「ブランド体現賞」を月次で表彰（社員×顧客×CMO三者選定）

### 8.9.2 事業部レベル

- 事業部のBrand KPIが目標達成した場合、事業部予算の柔軟運用権を付与

### 8.9.3 全社レベル

- 年次ブランドリフト調査でTier目標達成時、全社員へのインセンティブ
