## 13.3 着座率改善 オペレーション標準化【提案】

### 13.3.1 現状の課題（2026/4月次サマリより）

- 返信数199件 → 面談獲得73件（37%） → 面談実施53件（27%）
- 返信→実施までの離脱が約3分の2
- ターゲット外（50代以降、30代以降未経験）の返信が多い
- 後追い不足（架電後追いがOPS化されていない）

### 13.3.2 改善OPS（オペレーション標準）

```
[返信受領（媒体内チャット）]
   ↓ 30分以内
[一次架電]
   ├─ 通話成立：面談予約 → 自動LINEで詳細送付
   └─ 通話不成立：媒体内チャットでLINE誘導
       ↓ 1時間以内
   [LINE誘導再アタック]
       ├─ 反応あり：面談予約
       └─ 反応なし：媒体内チャットで「24時間以内に再ご連絡」予告
           ↓ 24時間後
       [二次架電]
           ├─ 通話成立：面談予約
           └─ 通話不成立：3日後に三次架電
               ↓ 3日後
           [三次架電]
               ├─ 通話成立：面談予約
               └─ 不成立：「ご都合のいい時間帯」アンケート送付
                   ↓
               [反応によりクローズ判断]

[面談予約完了]
   ↓ 24時間後
[前日リマインド：LINE + メール + SMS]
   ↓ 当日10分前
[当日リマインド：LINE]
   ↓ 不着座時
[即日リカバリー：当日中に再アタック]
```

### 13.3.3 着座率KPI目標

| 段階 | 現状（2026/4） | 目標（Month 12） |
|---|---|---|
| 返信→面談獲得 | 37% | 60% |
| 面談獲得→面談実施 | 73%（53/73） | 90% |
| 全体（返信→面談実施） | 27% | 55% |

→ Month 12時点で、同じ返信数なら面談実施数が2倍になる試算。

## 13.8 料率交渉 シナリオ別対応スクリプト【提案】

### シナリオ①「料率を下げてほしい」

```
「ご要望ありがとうございます。
 弊社の料率は業界標準値の30-35%帯に設定しています。
 これは『質と再現性』で勝負させていただくための
 適切な対価と考えています。
 
 Salesforce事例化企業として、
 候補者一人ひとりの伴走密度を保つには、
 適切な料率での取引が必要です。
 
 ただ、長期的な取引前提でしたら、
 入社者数◯名超で料率の段階的な見直しは
 ご相談可能です」
```

### シナリオ②「他社はもっと安い」

```
「他社さんの料率を下げて、その分の伴走密度を諦めるか、
 弊社の料率で、Salesforce事例化レベルの仕組みと密度を取るか、
 これは戦略的な選択かと思います。
 
 弊社が提供する『3年定着率』は業界平均より高く、
 採用1人あたりのトータルコストで見れば、
 長期的にはお得になる試算もあります」
```

### シナリオ③「成果報酬じゃなくて月額固定はできるか」

```
「弊社は成果報酬モデルが基本です。
 ただ、年間採用予定数が一定以上で、
 一定の戦略パートナーシップを結ぶ場合は、
 RPO型の月額固定+成功報酬の組み合わせも
 ご相談可能です」
```

### 料率交渉の原則

- 値下げは「条件付き」（量・期間・パートナーシップレベル）
- 単発の値下げは原則しない
- 法人営業マネージャーの判断を必ず通す
- 値下げした場合は、料率テーブルに記録
